Anonymous 20-Jul-20 02:27 PM
@Anonymous No, it is not needed. The CNOT gate checks the state of the control qubit and acts on the target qubit in the following fashion: - if the control qubit is at state |0>, do nothing; - if the control qubit is at state |1>, act as a NOT gate (that's where the name comes from); In the code I've sent above, the qubits q[0] and q[1] are initialized in the state |0>, as default. So, if q[0] is in state |0>, q[1] = |0> will not be altered, and then we have our first entry to the truth table: 1. |0> ∧ |0> = |0> If q[0] is in state |0>, but q[1] = |1>, again the result is do nothing to q[1], and the result is our second entry to the truth table: 2. |0> ∧ |1> = |1> Now, if q[0] is in the state q[0]= |1>, the cx gate will act as a NOT gate to the state of q[1]. The two possibilites are our last two entries to the truth table: The qubit q[1] is in state |0>, and the effect of the NOT gate to it will be to land it in state |1>, hence giving us the state q[1] = |1>, so: 3. |1> ∧ |0> = |1> And for the last entry in our truth table, the qubit q[1] is in state |1>, and acting a NOT gate in it will furnish us the state q[1] = |0>, thus: 4. |1> ∧ |1> = |0> This completes the truth table to have these four entries: 1. |0> ∧ |0> = |0> 2. |0> ∧ |1> = |1> 3. |1> ∧ |0> = |1> 4. |1> ∧ |1> = |0> Which is exactly that one from the XOR gate, which only returns True when only one or the other input is True.
@Anonymous but keep in mind that if you act the Hadamard gate to both qubits, you'll essentially translate them into the base of |+> and |-> states. That being said, if you act a Hadamard gate to each qubit, then apply the CNOT (to whom I've referred as cx in the comment above) to the second qubit controlled by the first one, and again apply the Hadamard gate to both, you'll get the same as having the second qubit being a control for the qubit 1 in a CNOT gate.
