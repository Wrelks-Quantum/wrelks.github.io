@Anonymous someone explain to me why the output of y is |y⊕f(x)> and not just |f(x)>
@Anonymous That's usually how quantum functions are designed – they take in an input register x, an output register y, and on their output side x is untouched while the output register is in state |y⊕f(x)>. The most common use for the register y is to start off with it in the all 0 state |00...0>, and so eventually you just get |00...0 ⊕ f(x)> = |f(x)>. Internally, the way this is done is usually by using CNOT (C-X) gates to "copy" the computation of f(x) onto each of the bits. This gives it the behavior of addition modulo 2 on the output register. For an example you can take a look at some of the exercises in https://qiskit.org/textbook/ch-ex/ex1.html
Set 1. Classical Logic Gates with Quantum Circuits
A university quantum algorithms/computation course supplement based on Qiskit
You'll notice that sometimes, the ability to have the output be the state |y⊕f(x)> is actually useful, more so than if you only got the state |f(x)> as output – take a look at the Deutsch-Josza algorithm, which makes use of this property.
