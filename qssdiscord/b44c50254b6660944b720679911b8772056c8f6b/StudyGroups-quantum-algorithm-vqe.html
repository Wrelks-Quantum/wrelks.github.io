Anonymous 29-Jul-20 12:59 PM
Yeah, I find this confusing too, but remember that an expectation value <psi | P |psi> is the average value you get from performing a measurement on |psi> -- you don't need to separately make an additional copy <psi| or anything. In terms of what you posted, |psi> = U|00...0>. Note that U is included in that definition, so since you don't need a separate copy of <psi|, you don't need to explicitly apply that U^dagger . You just start with |00...0>, apply U, and perform the correct rotations on each qubit so that you end up measuring the desired Pauli string P. Did that make sense?

Anonymous 30-Jul-20 06:44 AM
@Anonymous Yes. For example suppose that psi is only one qubit and the operator is Z. Then you create psi as you said, then measure it many time times. Each measurement result is 1 or -1, and their average is <psi|Z|psi>.
You can easily verify that for psi=alpha|0>+beta|1>, <psi|Z|psi> is indeed |alpha|^2-|beta|^2, which is the average of the measurement results.
Now similarly if the operator is X then you create psi, apply a Hadamard gate on it, and measure.
The same goes for multiple qubits, and you do it separately for each term in the sum, as you said.
